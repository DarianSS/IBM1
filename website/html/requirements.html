Requirements

Description Type Category Priority

Use cases

Use Case

LoginAttempt

ID

IBM1

Brief Description

A login attempt from unusual location

Actors

Clients ; Third party

Preconditions

User must be using the mobile app

Main Flow

1.    System receives a login attempt from a location different to the mobile phone’s current geolocation

2.    User gets notified about the third party authentication attempt on his mobile app

3.    User then receives an option to temporary block transactions

Post Conditions

Temporary transaction block is set up to prevent fraud

Alternative flows

None

 

Use Case

TransactionDisplay

ID

IBM2

Brief Description

Displaying the recent transaction on the map

Actors

Clients

Preconditions

User must be using the mobile app

Main Flow

1.    User makes a transaction

2.    System identifies the spending and records the geolocation

3.    System uploads the location on the map

4.    User opens the ‘Map’ tab

5.    System displays the transactions geolocation on the map

Post Conditions

A transaction map is updated with a recent spending

Alternative flows

TransactionMapBackup

 

Use Case

LimitTransaction

ID

IBM3

Brief Description

Setting up a transaction limit

Actors

Clients

Preconditions

User must be logged into the banking system

Main Flow

1.    User taps on the transaction limit link

2.    System displays an input tab

3.    User chooses desirable limit of daily spending

4.    System sets the limit on the amount of money that can be spent daily

Post Conditions

A limit for daily spending is set up

Alternative flows

RemoveLimit

RemoveLimit

ID

IBM4

Brief Description

Removal of the daily spending limit that has been previously set

Actors

Clients

Preconditions

Daily spending limit must be set up

Main Flow

1.    User opens the limit tab

2.    System prompts an option to remove or change the limit

3.    User selects limit removal and confirms the selection

Clients status has been updated and limit is no longer present

Alternative flows

None

 

Use Case

WebAuthenticationError

ID

IBM5

Brief Description

An error which occurs when user logs via SMS method

Actors

Clients

Preconditions

None

Main Flow

1.    User inputs his phone number

2.    System sends out a pin code to the phone

3.    User enters the code incorrectly

4.    System denies login attempt and makes SMS method no longer available

Post Conditions

Next login attempt cannot be done via SMS

Alternative flows

None

 

Use Case

TransactionMapBackup

ID

IBM6

Brief Description

Transaction map update when phone was not connected to the internet

Actors

Clients

Preconditions

User must be logged into the mobile app

Main Flow

1.    User performs a transaction

2.    The app cannot extract information due to no internet connection

3.    User gets access to internet connection later on

4.    System updates the map accordingly by extracting transaction details from the bank account

Post Conditions

The map is updated

Alternative flows

None

 

Use Case

MobileAuthentication

ID

IBM7

Brief Description

User logs into the system with the aid of machine learning algorithms

Actors

Clients

Preconditions

User has to access the system at specific time and from a specific network

Main Flow

1.    User runs mobile app at a certain time period

2.    System compares current login time with frequent login times

3.    System requests access confirmation

4.    User confirms instant access

Post Conditions

User is logged in to the bank account

Alternative flows

WebAuthenticationOption1 ; WebAuthenticationOption2 ; FailsafeOption
